来看一看常见的排序算法
Posted on 2020-07-08   Updated on 2020-12-12 GO Sort

sort

sort是计算机算法中璀璨的明星，sort在我们的计算机中有着不可取代的地位，可以这样说如果失去了高效的sort算法，我们的计算机就不会发展到像今天这样精彩。sort已经是计算机行业的基石算法类别之一，这个仅仅只是将数据排序的小小算法又是经历了一代又一代人的努力才发展成今天这样子，sort算法多种多样，每种的思想不一样，时间也不一样，但是却都是那么的优雅，值得我们去学习。

sort也是面试中常问的题目，这是一个程序员非常基本技能，如果你口口声声算法但是却不能实现常见的经典的算法那可就让别人不是那么的信服了，今天抽空整理了一下常见的sort算法。这些名字也是我模糊记得的，可能有的不这么叫，但是这个咱们不计较啊！

这些算法均用GO语言实现，并且给出Test案例。

冒泡排序

最基本的一种算法，有着最好听的、最容易被人记住的名字

将像它的名字一样，冒泡是最适合用来解释这个算法的思想的名词

选择排序

这是人脑很容易理解的算法，不断选出最小的那一个放到前面，最后整个数组也就有序了

步骤：

  • 从i开始
  • 遍历i之后的数组，寻找最小值
  • 这个最小值放到最开始的位置

私以为这是人脑最容易理解的sort算法

插入排序

插入排序就是我们从数组的最开始的位置遍历，每走都一个位置就把该位置的a[i]插到之前的合适的位置，因为我们是从0开始的，那么实际上我们就是保持了[0, i - 1]的位置，第i个数插在哪里呢？

想像我们一直向前走，因为前面是有序的，那么一旦遇到一个小于或者等于a[i]的数就停下来，这样就变然使得[0, i]也有序

其实你的任务就是一个，[0, i - 1]有序，你应该把第i个数放在哪里？

Shell排序

在插入排序的基础上进行改进，有了从宏观出发的考虑

相一想插入排序的时候什么时候最烦？是不是这种情况？2， 3， 4， 5， 6， 1，此时你要1移动到最前面，是会特别慢

你最喜欢什么情况？一定是这样1， 2， 3， 4， 5， 6，这时候你在放置6的时候一下子就知道比5大，不动就可以

Shell的哲学就是：

  • 我先大范围调动，这样调动之后不一样有序，但至少比之前有序，即使是最坏的情况也不影响，因为调度的范围比较大
  • 最后小范围调动，这时候应为顺序性相对较好，很大几率会碰见较好的情况

归并排序

归并排序可以算是高端的排序算法了，不仅仅是有高端的名字，更重要的是有了高端的性能，这是排序算法的分水岭

博主之前在leetcode的那篇博客中间提到过递归的能力，现在是考验递归的时候了，来一起想一下，给出归并排序的步骤：

  • 将数组左边部分变成有序
  • 将数组右边部分变成有序
  • 把两个有序数组归并成一个有序数组

第三步如果单独拿出来大家其实都会，就是扫描一遍就行了

有人可能会想左右是怎么变得有序的呢？

哈哈，其实这样想你就是没有理解递归的精髓，我说过递归要专注于当下，你本身就是写的排序算法，调用自己不就行了？ok，那这能保证正确性吗？

这完全取决于你当前这一步，就看你会不会写把两个有序数组合成一个有序数组，并不困难！

堆排序

堆排序需要构建最大堆或是最小堆，这个需要一些理论的支持，大家可以深入去看一下

另外需要注意的是我们并没有用到数组的0这个位置，从1开始可以帮助得到父子节点的关系为p = c * 2或者p = c * 2 + 1，这个结论任何时候都成立，不必单独考虑0处的特殊情况

快速排序

这个名字已经代表了快排的地位，sort的目标是什么，不就是快吗？而大名鼎鼎的Quick Sort可以说是sort算法中登峰造极的算法

你完全可以用你想得到的词汇来赞美快排，它真的值得

快排的思想本质也是简单的：

  • 对于一个数组，构造一个位置p，是p左边的数不大于p处的数字，右边的不小于（这样p的位置必然是最后有序之后应该在的位置）
  • 对于p左边的部分做同样处理
  • 对于p右边的部分做同样处理

思考一下为什么不是左边的数小于p处的数字，右边的大于，因为这种条件过于严苛，以我们从两边相中间靠拢的方法来说，如果遇到相等的情况就不好处理，换句话说，如果有多个与p处相等的数字，你根本就不能找到一个数字是它左边的数字小于它，右边的大于它

即使我们不考虑相等的情况，相等的数字也会在进一步的处理中靠过来

Update

对于常见的排序算法，我们的前提都是完全不知道任何数据的性质，只有用<关系进行比较排序，所以时间最多只能优化到O(logN)。但是往往有一些数据是有一定的性质的，例如高考分数都在[0, 750]的区间、等长字符串的长度相等，利用这些性质我们可以开发出更好的排序算法！

桶排序

因为我们知道数据的区间，例如array中的数据都在[0, max)之间，我们就可以直接设置一个bucket[max]数据，bucket[n]表示n有多少个，最后我们遍历bucket数组，顺序输出index就是排序之后的array！

基数排序

如果某个array中的数字都是n位数，例如都是三位数arr = [102, 111, 321, 908]，我们就可以从最低位开始，依次按照每一位的数据排序。这里我们也要设置一个桶数组bucket[radix]，用于记录按位排序的情况，当所有的位都完成排序之后，整体的数组就是有序的！

计数基数排序

在基数排序的基础上，不用桶去记录每个位上的结果，而是用一个count数组记录相关为上的数字个数，然后计算出偏移位置。

例如count = [1, 2, 0, 2, 1, 3, 1, 2, 4, 0]，可以计算出offset = [0, 1, 3, 3, 5, 6, 9, 10, 12, 16]，表示当该位为idx是，偏移的位置为offset[idx]，然后在将offset[idx] + 1，这样就得到了下一个的位置！
