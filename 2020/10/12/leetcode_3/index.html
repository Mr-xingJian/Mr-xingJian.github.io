8.leetcode-198. 打家劫舍（2020.10.16 14:12）

这种题目看到的第一眼应该就是DFS暴力递归求解，但是因为考虑到时间的限制，暴力肯定不能通过所有测试用例。

这里采用的是动态规划的解法，从后至前计算第i个到最后一个所能获取的最大财富，然后利用已经得到的值去求解。在求解到dp[i]的时候，判断i+1有没有去过来综合求解，主要的动态规划表达式如下：

  • 如果i+1去过：dp[i] = max(dp[i + 1], nums[i] + dp[i + 2])
  • 如果i+1没去：dp[i] = nums[i] + dp[i + 2]

2021.01.05 09:40

今天在做打家劫舍II的时候发现了一个更简单的方法，我们不需要整个dp数组，我们仅仅需要后两个的结果就行了，代码更加简洁，边界条件也没有之前复杂了！

19.leetcode-174. 地下城游戏（2020.12.18 13:55）

本质是一个动态规划的题目，设想我们首先不考虑一个矩阵，而是考虑一个线性的array，并且我们按照从后至前的顺序考虑。

首先我们想怎么确定这个最少的血，很自然的就是想到从前到后累加，看看整个过程中出现的最小值，如下：

[3, -8, 5, 6, -9]，从前至后累加[3, -5, 0, 6, -3]，此时出现的min = -5，那么勇士最少带6滴血出门。

然而这只是静态的变化，如果这个序列是不断变化的怎么办？

例如，此时在序列[……]前面插入一个x会有什么影响: [……] => x [……]：

加入我们已经知道原序列的累加最小值为tmin，试着讨论一下：

  • x > 0 && tmin > 0 && x > tmin(x = 9, tmin = 2): new_min = x = 2
  • x > 0 && tmin > 0 && x < tmin(x = 7, tmin = 9): new_min = x = 7
  • x < 0 && tmin < 0 && x < tmin(x = -7, tmin = -2): new_min = x + tmin = -16
  • x < 0 && tmin < 0 && x > tmin(x = -7, tmin = -9): new_min = x + tmin = -16
  • x > 0 && tmin < 0(x = 7, tmin = -2): new_min = x + tmin = 5
  • x < 0 && tmin > 0(x = -7, tmin = 2): new_min = x = -7

这样就十分清楚了，比一般的动态规划复杂一点，有了这个之后你就能思考一下，怎么去从右下角到左上角填写dp二维数组了！

因为这一题我们需要在原地修改，那么就不能直接将0改为1或者是将1改为0，而是将由生到死的变为2，由死到生的变为3，这样的话就能通过当前的细胞状态判断这个细胞之前的状态，规律如下所示：

  • 1: last = 1

45.leetcode-238. 除自身以外数组的乘积（2020.12.29 11:30）

这一题也是在空间和时间上有一定的限制，我们可以利用输出数组做一定的文章，首先用输出数组记录从右至左的乘积，例如[1, 2, 3, 4, 5]可以得到[120, 120, 60, 20, 5]，有了这些条件之后我们再去思考怎么进行下一步。

显然对于nums[i]如果我们知道左边的乘积left和右边的乘积right就好办了，有部分的乘积在ret中记录，left我们可以在遍历的时候不断累乘，这样就能得到了，看看这个示例：

nums = [1, 2, 3, 4, 5]，计算得到ret = [120, 120, 60, 20, 5], left = 1，接下来对nums进行遍历：

  • i = 0, ret[0] = left * ret[1] = 120, left *= nums[0] = 1
  • i = 1, ret[1] = left * ret[2] = 60, left *= nums[1] = 2
  • i = 2, ret[2] = left * ret[3] = 40, left *= nums[2] = 6
  • ……

这样就能得到输出数组ret，并且是O(n)的时间！

57.leetcode-292. Nim游戏（2020.01.04 11:02）

这一题也是比较巧妙，我当时就其实是一步一步推到出来的，推导的过程也是很简单的。首先，我们很容易知道前几个n的情况(结果中1表示获胜)：

石头：1 2 3 4 5 6 7 8 9 ……

结果：1 1 1 0 1 1 1 0 1 ……

其实很容易发现规律，但是规律并不能保证我们一定是对的，可以证明一下。

首先如果只有4块石头，那么先手必然是输的。后面的5,6,7，我们都能将对手送到4块石头的局面，那么我们就赢了，到了8块石头，无论我们先手拿几块石头，对手都可以让我们处于4块石头的局面，此时我们必输。

这样后面的9,10,11都可以这样递推，以8块石头为基准，我们要做的就是让对手面对8块石头的局面。

这样我们就不难判断自己的猜测是正确的，推理也可以证明。

58.leetcode-213. 打家劫舍II（2020.01.05 09:42）

这一题真的绕了好大的一个圈子，刚开始我想的是记录最后一个元素有没有使用，然后去判断第一个元素到底用不用，这样导致最后十分的复杂，前几次提交一直没做出来。

昨天因为状态不太好，就没有硬刚，今天突然恍然大悟，既然是一个环，我们只要将其拆成一个链不就可以使用之前的方法完成吗？

头尾两个元素必然不能同时使用，所以有如下结论：

rob([0, 1, …, n]) = std::max(robAux([0, 1, …, n - 1]), robAux([1, 2, …, n]))

robAux就是我们之前对首位不相连的打家劫舍问题的求解方法！
