GO中的数据类型
Posted on 2020-06-02   Updated on 2021-02-26 GO

记录一下Go语言的学习经历

GO中的数据类型（data type）

基本数据

整型

int

在go中，整型分为有符号和无符号整数，分别有一下几种：

  • int，int8，int16，int32，int64
  • uint，uint8，uint16，uint32，uint64

其中int和uint会根据硬件平台选择宽度，我们无法假设（不同编译器在相同硬件平台编译的结果也可能会不相同），其余均是指定长度的。

rune

rune与int32等价（只是一个别名），表示一个Unicode码点

byte

byte与uint8等价，强调数值是一个原始数据而不是小整数

uintptr

uintptr没有指定具体的bit大小但是足以容纳指针，底层编程会用到

要点：

  • go中的类型不会自动转型，int32与rune之间也需要强制转性，int与int32亦是如此

  • 在go中%运算符的结果取决于被取模数的符号

  • bit溢出的部分会被丢失（无符号数左移用0填充，有符号数用符号位填充，即有符号数在移位时要注意）

    • 正数左移溢出

    • 负数右移

    • 负数也会出现溢出现象

    • 注意，所有的移位操作都是在补码的基础上进行，用源码有时候无法直观展示

  • 几个重要的位运算符

    • &
    • |
    • ^ 这个可以充当亦或运算符，单独使用作为取反运算符
    • &^ 位清空（z = x &^ y，如果y的bit为1，则z置零，否则等于x对应的bit）
    • << 左移
    • >> 右移

  • %b可以按位打印二进制数

  • %d，%o，%x分别可以打印十进制，八进制和十六进制的数字

  • %c表示打印字符，%q表示打印带单引号的字符

  • %T可以打印出类型信息

浮点数

浮点数在go中表示为float32和float64，相对于其他语言，没有double，算数规范由IEEE754标准支持

要点：

  • 最大值：math.maxFloat32和math.MaxFloat64，最小值无限接近于0

  • float32大约支持6个十进制的精度，float64支持大概15个十进制精度

  • 打印参数有%g，%e，%f

  • 浮点数中的特殊值，正负无穷大和NaN

复数

go提供了两种复数类型，分别时complex64和complex128，分别对应float32和float64的精度

要点：

  • compelx()内置函数用于构造复数
  • real()和image()用于返回相应的实部和虚部
  • cmplx是内置的复数处理伴生包

布尔

bool只包括true和false

要点：

  • 区别于C语言。go中的bool更加规范严谨，不会自动隐式转换为1和0

  • 可以将转换封装一下

字符串

在go中，字符串是不可改变的字节序列，文本字符串通常被解释为采用UTF8编码的Unicode（rune）码点的序列，显然这里解决了笔者的一个错误的想法，Unicode是对字符的码点表示，UFT8是将码点转为相应编码格式的编码方式，二者并不是同一层次的概念。按照概念来讲Unicode和ASCII这种是同一个层级的概念（字符集），而UTF8、ISO和GBK是一个层次的概念（编码方式）

字符和字节并不是一一对应，Unicode中的一些较大的码点是不能用一个字节表示的

要点：

  • string可以进行切片操作
  • string也可以直接进行+运算
  • ==和<也是可以的，是通过逐个字节比较完成的
  • 利用字符串产生的切片也是使用的字符串的底层数组
  • s[idx]=’a’这种赋值方式在go中是不被允许的
  • Unicode码点也可以直接写进字符串

原生字符串：

  • 没有转义操作，全部都是内容
  • 使用反引号

带一下字符集与编码方式的知识：

字符集：用数字表示字符的一套映射方式

编码方式：实现相关数据到字符的映射

Unicode：几乎涵盖了100多种语言的字符，收集超过120000个字符

显然使用UTF32（rune）可以完整的表示相关的码点，但是存在大量不常用的码点，会导致空间的浪费，于是又出现了UTF16，但是依然不够灵活

UTF8：使用一到四个字节表示码点

  • 缺点：不等长，不能按照索引获取字符
  • 优点：
    • 节省空间，灵活，完全兼容ASCII，不需要有结尾标识符
    • 统一之后，一些string操作不用解码和编码

当然，在程序运行的时候，我们还是希望数据保持整齐，我们会更加期望使用rune进行处理，那么就需要通过强制转换将string转成[]rune数组([]rune(string))，同时如果需要进行string写入到文件的话，可以利用string([]rune)将Unicode码点数组转成string

对于string类型的操作同时也离不开四个工具包：

  • unicode
  • bytes（bytes.Buffer用的很多）
  • strings
  • strconv

常量

go中常量表达式的值在编译器计算，而不是拖到运行期，常量的所有算术运算和比较运算的结果也是常量

常量的声明方式：

  • 批量声明：只保留第一个右表达式即可

  • iota常量生成器：生成一组相似规则的初始化常量

无类型的常量：go中许多常量没有明确的类型，编译器为没有明确的基础类型的数字常量提供更高精度的算数运算，至少有256bits，且无类型的数据不需要经过转型就可以运算（这种转化要求是合法的）

分类：

  • 无类型的布尔
  • 无类型的整数
  • 无类型的字符
  • 无类型的浮点
  • 无类型的附属
  • 无类型的字符串

假如上述的1 << iota一直延续，肯定会超过int64，但是go也是支持的

复合数据

复合数据是在基础数据的基础之上组合而成的更复杂、语义更丰富的数据类型

数组

数组 = 元素类型 + 长度

长度是固定的，不同长度数组不能直接赋值，长度相同的可以

也正是因为数组长度是固定的，那么长度必须是常量或者利用…直接推导

要搞清楚一点的是，数组是一种值，那么无论是赋值还是进行传参都是拷贝

要点：

  • 数组元素可比的话数组也可以比较
  • 数组因为长度不可变，使用有局限性，但是在一些例如hash生成固定长度的消息摘要时比较适合
  • 函数传参时如果要改变数组就要传数组的指针

slice

切片[]T，是一组动态可变的相同类型元素的集合，相比于数组，silce具有动态扩展的功能

组成部分：

  • ptr：指向底层数组的切片开始指针
  • len： 当前可见程度
  • cap：在使用原来的底层数组（不重新分配内存）的情况下所能扩展的最大长度

切片中的len限制了可见性，cap限制了切片的范围，在切片的时候如果超出了cap会抛出一个panic，但是如果只是超出了len，那么意味着扩展了切片，但是扩展的元素的值就是由底层数组来决定的了，string的切片也是共享原string的底层数组，在不涉及内存重新分配的时候，这种slice可以在常数时间完成

要点：

  • slice是不能直接比较的，bytes.Equal()是系统提供的比较[]byte切片的函数

  • 如何生成一个nil的slice（底层没有数组，len和cap都是0）

  • make函数也可以创建相应的slice，make在底层创建了一个匿名数组

slice扩容机制：slice使用append函数进行扩容

  • 为了保证扩容的时间，每次扩容并不是只增加一个，而是扩展到之前的两倍，防止频繁扩容复制导致效率降低

  • 因此slice中的ptr是有可能会改变的，所以append函数需要返回一个新的切片保证正确性（当然ptr不是必然会改变）

map

map是go中的哈希表，存储相关的k、v键值对

要点：

  • k必须支持==运算，但是最好不要是float，浮点因为在计算机中的表示问题，可能相等并不精确

  • 从map中取出不存在的k的v会返回zero-value

  • delete这个内置函数可以删除相关的键值对，并且不必担心删除一个不存在的k

  • 你无法对map中的元素取址，&ages[“mike”]不合法，因为map的扩容可能会导致原来的地址失效

  • map中的元素遍历是无序的，保证了你的程序不会依赖特定的hash函数，如果你依赖特定的hash函数，虽然可能不会出现问题，但是这不稳定

  • 从map中取v的时候还会返回一个存在性的标志

struct

结构体是聚合数据，有任意多的类型聚合而成（类比C中的结构体）

go中的struct通过.操作符使用内部的数据，对于value和pointer都是如此，point会被自动的解析称为value（*）

不同于其他语言的公有私有，go语言根据成员name的首字母大小写定义自己的导出规则，这个规则也同样适用于包中的变量和函数，go耍了一点”小聪明”就干掉了Java中的三个关键字

结构体的zero-value是其中的每一个成员都是zero-value

结构体之间的比较性取决于结构体成员的比较性，如果struct可以比较，即可充当map中的key

go中的面向对象

go中的oop没有相关的继承，取而代之的是组合，这种组合分为具名内嵌和匿名内嵌：

  • 具名：内部成员都是有name的

  • 匿名：内部成员只有类型（或是类型的pointer），没有名字

  • 外层的结构体不仅仅是获得了匿名类型的所有的成员，也获得了匿名类型的所有方法集
